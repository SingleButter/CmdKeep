**把自己常用但记不牢的命令，变成一个可检索、可复制的小型知识库 App**。下面我给你一个在 Windows 上做这类 App 的“靠谱设计方案”（从功能到数据结构到技术选型。

------

## 你真正需要解决的核心问题

1. **快速找到**：我想做 “改 branch 名” → 立刻搜到 `git branch -m ...`
2. **少打字**：复制/一键执行（可选）比手敲省时
3. **能分类+标签**：Git / Linux / Docker / Azure / SQL……
4. **有上下文**：命令本身 + 说明 + 示例 + 注意事项（比如危险命令）
5. **可迭代**：未来你会不断加内容、改内容、做模板化

------

## MVP（最小可用版本）功能清单（建议你先做到这一步）

### 1) 命令条目（Command Entry）

每条保存这些字段：

- **标题**：比如「重命名本地分支」
- **命令**：`git branch -m old new`
- **说明**：一句话解释
- **分类**：Git / Linux / Docker ...
- **创建/更新时间**

### 2) 全局搜索 + 过滤

- 顶部一个搜索框：支持标题/说明/命令内容搜索
- 侧边栏分类

### 3) 一键复制

- 列表里每条命令旁边一个 Copy 按钮
- 复制后 toast 提示

### 4) 添加新的命令
- 右上角有一个按钮打开弹窗，让用户填写需要的字段之后将新的命令添加。
